## Default salinity units:
- Salinity has no units by default. Don't use psu. When setting up attributes on netcdf files use 'units':'1e-3' to indicate that the salinity values are in practical salinity units (psu) and not in absolute salinity units (g/kg).


## Terminal outputs and log files
- Whenever a command's output is redirected to a log file (i.e. not visible in the terminal), explicitly remind me of that.
- I usually run long jobs inside a tmux session, which keeps the process alive across detach/reattach. So the simplest pattern — live in the terminal **and** persisted to disk — is just `tee`:
  ```bash
  my_command 2>&1 | tee logs/run_$(date +%Y%m%d_%H%M%S).log
  ```
  Detach with `Ctrl-b d`, reattach later with `tmux attach`.
- Only fall back to `> log 2>&1 &` (or `nohup`) when I'm *not* inside tmux/screen and need the job to survive logging out. In that case tell me how to follow it live in another pane:
  ```bash
  tail -f logs/run_<timestamp>.log
  ```
- Do **not** pipe a long-running command through `| tail`, `| head`, or similar — those buffer until the upstream finishes, so I lose the live view. If you need to limit what *you* see in the tool output, run the command without the pipe (with `tee` if persistence is wanted) and then inspect the log file separately.
